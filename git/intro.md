
# Git

- 가장 대중적인 버전 관리 시스템.

# Concepts

- Repository : Git으로 트래킹되는 프로젝트 폴더.
- [Clone](clone.md) : 리모트 레포지토리를 로컬로 복사하는 작업.
- [Staging](staging.md) : Git에 변경 사항을 지정하는 작업.
- [Commit](commit.md) : 스테이징된 변경 사항을 스냅샷으로 저장하는 작업.
- [Branch](branch.md) : 독립적으로 관리되는 워크플로우.
- [Config](config.md) : Git 설정.
- [Checkout](checkout.md) : 특정 커밋 또는 브랜치로 이동하는 작업. 대체 가능.
- Origin : Git 프로젝트의 리모트 레포지토리.
- HEAD : 현재 브랜치를 가리키는 포인터. Detached HEAD일 경우 현재 커밋.
- Tracked File : 커밋 혹은 스테이징이 되어서 Git이 관리하고 있는 파일. 
- Fork : 다른 사용자의 레포를 내 계정으로 복사하는 것.
- Pull Request : 프로젝트 관리자에게 작업물의 머지를 요청하는 과정.
